# 样本库记录列表弹窗内联编辑设计

## 背景

样本库的 `记录列表` 已经改成：

- 主页面只显示 3 条预览
- 通过 `查看全部记录列表` 打开完整列表弹窗

这解决了主页面滚动过长的问题，但当前完整列表弹窗仍然只是“列表查看器”：

- 可以切换记录
- 可以打开详情
- 不能直接在弹窗内完成完整维护

对于连续整理学习样本的场景，用户仍然需要在：

- 主页面详情区
- 记录列表弹窗
- 多个二级编辑弹窗

之间来回跳转，操作链路偏长。

这次改动的目标，是把 `记录列表` 弹窗升级成一个小型工作台，让用户在同一个弹窗里完成：

- 选择记录
- 编辑四块内容
- 整条记录统一保存

主页面不再承担样本详情展示，相关查看和维护统一收敛到弹窗工作台里处理。

## 目标

把 `查看全部记录列表` 弹窗升级成支持完整内联编辑的维护工作台。

成功标准：

1. 弹窗内可以直接编辑一条学习样本的四块内容
2. 保存方式为“整条记录统一保存”，而不是分块保存
3. 用户可以在弹窗里快速切换记录并连续维护
4. 主页面不再展示具体样本详情，由弹窗统一承接查看与编辑
5. 不新增新的弹窗体系

## 非目标

本次不做以下内容：

- 不在主页面保留按记录展开的详情展示
- 不移除现有基础内容 / 参考属性 / 生命周期 / 预判复盘二级编辑弹窗
- 不新增后端接口
- 不把“误报案例”弹窗也一起升级成内联编辑器
- 不引入多记录同时编辑或批量保存

## 方案概述

### 1. 弹窗形态

继续复用现有 `sample-library-modal`，但把 `记录列表` 弹窗从“单列列表展示”升级成“左侧列表 + 右侧编辑器”的 master-detail 结构。

主页面上的样本区只保留：

- 当前筛选条件
- 3 条记录预览
- `查看全部记录列表` 入口

不再展示右侧详情区中的样本具体数据。

弹窗内部分为两栏：

- 左栏：当前筛选条件下的完整记录列表
- 右栏：当前选中记录的完整内联编辑区

保留当前筛选上下文：

- 过滤条件不变
- 集合筛选不变
- 搜索条件不变

因此用户在主页面上已经缩小到某个子集后，弹窗里看到的也应是同一批记录。

### 2. 左栏：记录列表

左栏继续使用当前记录卡片样式和摘要信息，保留：

- 当前选中高亮
- 当前筛选结果计数
- 点击切换记录

与之前不同的是，点击左栏记录不再直接关闭弹窗或跳转主页面，而是：

- 尝试切换当前编辑对象
- 如果当前记录没有未保存修改，直接切换
- 如果当前记录存在未保存修改，先进行放弃确认

### 3. 右栏：完整内联编辑区

右栏展示当前选中记录的完整编辑表单，覆盖四块内容：

1. 基础内容
2. 参考属性
3. 生命周期属性
4. 预判复盘

这些表单字段沿用现有四个二级编辑弹窗的字段定义和默认值逻辑，避免出现两套不一致的表单语义。

右栏不再只是摘要卡片，而是直接提供可编辑字段。

### 4. 保存模型

保存方式采用 `整条记录统一保存`。

即：

- 右栏所有字段先写入本地草稿状态
- 用户点击 `保存整条记录` 后
- 一次性组装完整 PATCH payload
- 单次提交：
  - `note`
  - `reference`
  - `publish`
  - `calibration`

这样可以确保：

- 用户把一条记录看作一个整体
- 不需要在四个区块之间反复点击保存
- 不会出现“前两块已保存、后两块还没保存”的心智割裂

### 5. 危险操作

`删除记录` 不并入整条保存动作。

删除应保持独立危险操作：

- 放在右栏头部或底部的独立区域
- 继续复用现有删除确认弹窗

原因：

- 删除和“保存草稿”属于不同风险等级
- 不应让用户在同一主按钮流程里同时承担编辑保存和删除判断

## 交互细节

### 打开弹窗

用户点击 `查看全部记录列表` 后：

- 打开 `record-list-inline-editor` 类别的样本库弹窗
- 左栏显示当前筛选结果的完整记录列表
- 右栏默认加载当前已选中记录
- 如果当前没有已选中记录，则回退到当前筛选结果里的第一条

### 编辑

用户在右栏输入、勾选、切换字段时：

- 只更新当前记录的本地草稿
- 不立即请求接口

需要支持的字段范围与现有弹窗保持一致，包括：

- 基础内容：
  - 标题
  - 正文
  - 封面文案
  - 合集类型
  - 标签
- 参考属性：
  - 是否启用
  - 参考等级
  - 备注
- 生命周期属性：
  - 发布状态
  - 发布时间
  - 点赞 / 收藏 / 评论 / 浏览
  - 平台原因
  - 回填备注
- 预判复盘：
  - 发布前预判
  - 风险等级
  - 预判表现
  - 置信度
  - 模型
  - 预判时间
  - 预判理由
  - 发布后复盘
  - 实际表现
  - 复盘时间
  - 是否命中
  - 是否建议进入参考样本
  - 偏差原因
  - 验证 / 被推翻信号
  - 规则优化候选
  - 复盘备注

### 辅助动作

预判复盘区仍保留现有辅助能力：

- `从当前检测预填预判`

但它只影响当前草稿，不自动保存。

### 切换记录

左栏点击另一条记录时：

- 如果当前草稿未改动，直接切换右栏内容
- 如果当前草稿已改动，弹出确认

确认文案固定为：

- `当前记录有未保存修改，是否放弃并切换？`

用户选择：

- `继续切换`：丢弃当前草稿并切换
- `取消`：留在当前记录

### 保存

点击 `保存整条记录` 后：

- 执行统一校验
- 若校验通过，则发起单次 PATCH

保存成功后：

- 弹窗保持打开
- 左栏当前记录卡片摘要刷新
- 主页面预览区同步刷新
- 当前选中记录保持不变
- 在弹窗内给出成功反馈

### 关闭弹窗

关闭时如果没有未保存修改，直接关闭。

关闭时如果存在未保存修改，也应进行一次放弃确认，避免用户误关。

确认逻辑与切换记录一致，但文案改为：

- `当前记录有未保存修改，是否放弃并关闭？`

## 状态设计

在现有 `appState.sampleLibraryModal` 基础上，为记录列表内联编辑模式维护以下状态：

- `kind`
- `recordId`
- `draft`
- `initialSnapshot`
- `dirty`
- `switchTargetRecordId`

语义如下：

- `recordId`
  当前右栏正在编辑的记录 id
- `draft`
  当前记录的本地可编辑草稿
- `initialSnapshot`
  当前记录打开或最近一次保存成功后的基线快照
- `dirty`
  当前草稿是否有未保存修改
- `switchTargetRecordId`
  当用户尝试切换记录但需要先确认放弃时，暂存目标记录

### 草稿来源

打开弹窗或切换记录时：

- 从 `appState.sampleLibraryRecords` 中找到目标记录
- 生成统一草稿对象
- 同时生成基线快照

### dirty 判断

任何字段变更后，需要重新比较：

- 当前 `draft`
- 当前 `initialSnapshot`

若两者不同，则标记 `dirty = true`。

## 校验规则

统一保存前复用现有分块校验逻辑，不新增一套新规则。

至少包括：

- 基础内容：
  - 标题 / 正文 / 封面 / 标签 至少有有效内容
  - 必须有合集类型
- 参考属性：
  - 启用参考样本时必须有参考等级
- 生命周期与预判复盘：
  - 沿用当前字段容错逻辑

如果校验失败：

- 不提交接口
- 在弹窗内给出明确提示
- 保持用户当前输入不丢失

## 技术实现边界

主要前端改动集中在：

- `web/app.js`
- `web/index.html`
- `web/styles.css`
- `test/success-generation-ui.test.js`

实现策略：

1. 为记录列表弹窗新增内联编辑 kind
2. 抽取“记录 -> 草稿”和“草稿 -> PATCH payload”的转换函数
3. 复用现有四块表单 markup 逻辑，尽量避免复制两套字段定义
4. 新增 dirty 检测与放弃确认逻辑
5. 统一保存后复用当前工作区刷新路径

不改后端接口。

## 风险与控制

### 风险 1：弹窗内状态过重，切换记录容易丢失输入

控制：

- 显式维护 `draft` / `initialSnapshot`
- 切换记录前强制做未保存确认

### 风险 2：内联编辑和现有二级编辑弹窗出现字段逻辑不一致

控制：

- 字段定义、默认值、校验规则尽量复用现有实现
- 避免复制粘贴出第二套业务规则

### 风险 3：统一保存 payload 组装出错，导致只保存部分字段

控制：

- 明确使用单次 PATCH，统一提交四块内容
- 针对 payload 结构补测试断言

### 风险 4：弹窗过高、可读性下降

控制：

- 使用左右分栏
- 右栏内部独立滚动
- 保留区块标题和分组感，不把所有字段压成一长串

## 测试重点

需要验证：

1. 记录列表弹窗渲染为左右分栏结构
2. 左栏展示当前筛选下的完整记录列表
3. 右栏展示当前选中记录的四块完整表单
4. 编辑任意字段后会进入 dirty 状态
5. 有 dirty 时切换记录会被拦截
6. 有 dirty 时关闭弹窗会被拦截
7. 点击 `保存整条记录` 时走统一保存路径
8. 保存成功后左栏摘要与主页面详情同步刷新
9. 删除记录仍走独立危险操作，不混入统一保存
10. 主页面不再渲染样本详情区中的基础内容 / 参考属性 / 生命周期 / 预判复盘数据
