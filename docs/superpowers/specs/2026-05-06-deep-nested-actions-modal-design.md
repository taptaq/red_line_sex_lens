# 深层操作弹窗化设计

## 背景

当前 `学习样本` 相关界面里已经承载了较多表单内容，尤其是两块区域开始出现“嵌套里再嵌套”的问题：

- `PDF 批量导入学习样本` 中，每张导入卡片内还包含一整块可展开的 `高级属性`
- `学习样本详情` 中，`参考属性`、`生命周期属性`、`预判复盘` 都在详情区内继续承载较长表单

这些结构在窄宽度下容易出现：

- 操作区域没有足够空间
- 展开后阅读路径过深
- 标签选择器、状态摘要、表单布局相互挤压
- 样式在不同内容长度下更容易错乱

用户要求将这类深层操作改成弹窗形式，以释放页面空间，并统一交互方式。

## 目标

把 `学习样本` 区中“嵌套层级深、编辑项长、低频但重要”的操作统一改成弹窗流，让主页面更像总览与入口，深层编辑放到独立弹窗中完成。

本次弹窗统一使用：

- 居中弹窗
- 独立 `取消 / 保存`
- 未保存前不落库
- 保存成功后回写主页面摘要

## 非目标

本次不做以下改动：

- 不调整后端数据结构
- 不合并现有保存接口
- 不把 `基础内容` 主编辑区弹窗化
- 不重新设计标签选择器交互逻辑
- 不引入新的页面级路由或多步向导

## 交互决策

### 1. PDF 批量导入

每张导入卡片保留高频基础字段：

- 文件名
- 标题
- 封面文案
- 正文
- 合集类型
- 标签
- 点赞 / 收藏 / 评论
- `确认导入`

原来的 `高级属性` 折叠面板移除，改成：

- 一个 `编辑高级属性` 按钮
- 一个简洁状态摘要，展示参考属性与生命周期属性当前状态

点击按钮后打开弹窗，弹窗内部包含两个分组：

- `参考属性`
- `生命周期属性`

弹窗内点击 `保存` 后，只回写当前导入卡片的草稿状态，不直接写入正式样本库；正式入库仍以卡片上的 `确认导入` 为准。

### 2. 学习样本详情

`基础内容` 继续保留在主页面，原因是它是最高频编辑区，也是记录识别的主视图。

以下三块改成“摘要卡片 + 编辑按钮 + 弹窗”：

- `参考属性`
- `生命周期属性`
- `预判复盘`

主页面只展示：

- 当前状态摘要
- 关键字段预览
- 最近一次结果标签
- `编辑` 按钮

用户点击后进入对应弹窗，在弹窗中完成这一块的编辑与保存。

### 3. 保存方式

用户已明确选择：每个弹窗独立 `取消 / 保存`。

对应规则如下：

- `取消`：关闭弹窗，丢弃本次未保存改动
- `保存`：调用该模块原本对应的保存逻辑
- 保存成功后：
  - 关闭弹窗
  - 刷新对应页面摘要
  - 保持当前记录定位不变

## 前端结构调整

### 新增通用弹窗骨架

前端需要一个统一的弹窗容器，用于承载：

- 遮罩层
- 标题区
- 内容区
- 底部操作区

同一套骨架复用于：

- 批量导入高级属性编辑
- 学习样本详情的参考属性编辑
- 学习样本详情的生命周期属性编辑
- 学习样本详情的预判复盘编辑

### 内容来源

为了减少重复实现：

- 批量导入弹窗使用当前导入卡片草稿数据初始化
- 学习样本详情弹窗使用当前选中记录数据初始化
- 保存时复用现有的字段规范化与提交逻辑，不新增并行数据流

## 视觉与布局

弹窗样式应贴合当前产品色调，不使用浏览器默认裸弹窗观感。

约束如下：

- 弹窗主体宽度偏紧凑，不做超长横向表单
- 分组标题与说明清晰
- 底部操作区固定，避免长表单时按钮跑出可视区
- 移动端和窄屏时弹窗改为接近全宽，但仍保留边距
- 标签相关控件在弹窗中若出现，仍沿用当前统一标签组件样式

## 错误处理

### 批量导入弹窗

- 保存失败时保留弹窗，不清空用户输入
- 在弹窗内显示错误信息
- 不影响当前导入卡片的其他已填写基础字段

### 学习样本详情弹窗

- 保存失败时保留弹窗与输入内容
- 不切换当前记录
- 不推进或回退当前步骤状态

## 测试范围

需要补或调整的测试应覆盖：

1. 批量导入不再依赖 `高级属性` 折叠区，而是暴露弹窗入口
2. 学习样本详情的 `参考属性 / 生命周期属性 / 预判复盘` 暴露独立弹窗入口
3. 弹窗中存在 `取消 / 保存` 的独立交互
4. 保存后摘要正确回写，取消不会污染当前页面状态
5. 现有批量导入确认导入流程、样本详情保存流程不回归

主要改动范围：

- `web/index.html`
