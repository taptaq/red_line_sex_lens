# 风格画像查看与手动编辑设计

## 目标

恢复当前风格画像的可见性，并允许在前端直接查看、编辑当前生效画像，同时保留现有“参考样本变化后自动沉淀”的机制。

在最终实现中，画像生成策略进一步升级为“模型优先 + 本地规则兜底”，并与当前参考样本池保持实时同步。

## 设计决策

### 1. 展示入口

不恢复旧的独立画像面板，也不新增新的大型工作区。

采用现有通用弹窗模式：

- 在生成工作台区域增加一个“查看 / 编辑当前风格画像”入口
- 点击后复用 `sample-library-modal` 展示当前画像
- 展示内容包括主题、名称、风格摘要、偏好标签、避免表达、生成指导、来源样本

这样能和当前“长列表尽量收进弹窗”的交互方向保持一致。

### 1.1 数据同步口径

风格画像的数据源必须和“参考样本池”使用同一套最新口径，不再使用“所有启用参考属性记录再按权重筛选”的旧逻辑。

新的口径：

- 风格画像只基于当前参考样本池中的样本生成
- 打开风格画像弹窗时，先按当前最新参考样本池即时刷新一次画像
- 画像展示中的“来源样本数”必须和参考样本池当前数量一致

这样可以避免画像显示为旧快照、数量与样本池不一致的问题。

### 1.2 生成链路

风格画像不再只依赖本地规则汇总，而是先走模型总结，再由本地规则兜底。

最终链路固定为：

- `qwen`
- `kimi`
- `deepseek`
- `本地规则`

行为约定：

- 先基于当前参考样本池构建一份本地规则基底画像
- 再按顺序尝试使用 `qwen -> kimi -> deepseek` 输出结构化画像字段
- 一旦某个模型返回有效结果，就将该结果合并到本地规则基底上作为当前画像
- 若模型全部失败，回退到本地规则基底，并记录本次回退信息

这样既能优先利用模型总结能力，也能保证在无密钥或模型异常时仍然稳定可用。

### 2. 自动沉淀与手动编辑共存

手动编辑不能在下一次自动沉淀后被覆盖，因此采用“自动画像 + 手动覆写”模式：

- 自动沉淀继续根据参考样本刷新画像基底
- 手动编辑只作用于允许人工修订的字段
- 下次自动刷新时，来源样本、时间戳等自动字段继续更新
- 已手动修订的可编辑字段继续保留

可编辑字段：

- `topic`
- `name`
- `titleStyle`
- `bodyStructure`
- `tone`
- `preferredTags`
- `avoidExpressions`
- `generationGuidelines`

非编辑字段仍由系统维护：

- `id`
- `status`
- `sourceSampleIds`
- `createdAt`
- `updatedAt`

### 2.1 展示增强

风格画像弹窗中的来源展示需要从“仅显示样本 ID”升级为“优先显示样本标题，ID 作为辅助信息”。

同时补充：

- 更新时间使用本地可读格式，例如 `2026-05-08 15:51`
- 展示当前生成方式，例如“通义千问 / Kimi / 深度求索 / 本地规则汇总”
- 在模型回退时明确提示当前为“本地规则汇总（模型链路已回退）”
- 来源样本优先展示标题，同时保留样本 ID 作为辅助信息

### 3. API 边界

不恢复旧的公开画像 API。

新增仅供前端管理界面使用的 admin 路由：

- `GET /api/admin/style-profile`
- `PATCH /api/admin/style-profile`

同时在 `GET /api/admin/data` 中附带当前画像，方便前端统一刷新状态。

为保证弹窗始终拿到最新口径，`GET /api/admin/style-profile` 在返回前应基于当前参考样本池刷新一次画像，再返回最新结果。

该刷新过程中应：

- 使用当前参考样本池生成最新自动画像
- 重新应用已保存的 `manualOverrides`
- 返回附带 `sourceSamples` 和 `generationMeta` 的当前画像

## 错误处理

- 没有当前画像时，弹窗展示“尚未沉淀画像”，但仍允许保存主题/名称等人工初始化信息
- 保存时对数组类字段做去重与清洗
- 如果保存失败，弹窗内给出错误提示，不关闭弹窗
- 如果模型链路失败，仍应返回可用画像，并在 `generationMeta` 中记录回退状态与尝试链路

## 测试范围

- 手动覆写字段在自动刷新后仍保留
- admin 路由可读可写，旧公开路由仍保持 404
- admin 画像返回的来源样本数量与参考样本池一致
- 前端存在入口、弹窗动作和保存逻辑
- 前端展示来源标题、本地化更新时间和当前生成方式
- 模型链路按 `qwen -> kimi -> deepseek -> 本地规则` 顺序回退
