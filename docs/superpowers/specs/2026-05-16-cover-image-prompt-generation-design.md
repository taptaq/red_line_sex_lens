# 新增候选稿封面图 Prompt 生成设计

## 目标

在“新增候选稿”功能中，随正文、标题、封面文案、标签一起生成一条可直接用于出图的封面图 prompt，并在结果卡片中展示与复制。

这条 prompt 必须基于当前候选稿的标题与正文提炼封面元素，同时严格遵守模式差异：

- 短文模式：封面文案 + 不露脸的萌系宇航员形象 + 去掉手臂的国旗标识 + 高反差 + 吸睛
- 长文模式：露脸的萌系宇航员形象 + 去掉手臂的国旗标识 + 高反差 + 吸睛

## 设计决策

### 1. 产物形态

只生成一条最终封面图 prompt，不做多候选、不做正反双 prompt、不做二次点选。

生成结果中新增字段：

- `coverImagePrompt`

展示位置：

- 新增候选稿最终结果卡片内
- 标题建议为“封面图 Prompt”
- 提供“复制封面图 Prompt”按钮

这样可以把封面图生成和正文发布稿统一收口到同一张结果卡片里，避免新增一个独立工作流。

### 2. Prompt 生成原则

封面图 prompt 必须先理解当前候选稿，再输出视觉描述，不允许只输出空泛模板。

硬性要求：

- 明确引用标题和正文中的核心元素
- 必须体现“小红书封面感”
- 必须强调高反差、吸睛、视觉主体突出
- 必须去掉手臂上的国旗标识
- 不允许生成成人写实压迫感、杂乱背景、低对比弱主体等不符合封面使用场景的描述

模式差异：

- 短文模式必须包含“封面文案”这一层文案描述
- 长文模式不强制加入封面文案，但仍要突出封面级视觉钩子

推荐的 prompt 结构：

1. 主体：萌系宇航员
2. 状态：短文不露脸，长文露脸
3. 风格：高反差、吸睛、小红书封面风
4. 细节：去掉手臂国旗标识
5. 主题锚点：来自标题和正文的核心内容、情绪、场景或问题
6. 画面控制：主体清晰、背景简洁、不要杂乱、适合封面裁切

### 3. 生成方式

采用“模型优先 + 本地规则兜底”的方式，但与正文生成解耦：

- 先从候选稿的标题、正文、封面文案、标签中抽取封面元素核心
- 再由模型生成一条封面图 prompt
- 若模型失败或返回不合格文本，回退到本地规则拼装版 prompt

本地规则兜底至少要保证：

- 短文模式：`封面文案 + 不露脸的萌系宇航员 + 无国旗臂章 + 高反差 + 吸睛`
- 长文模式：`露脸的萌系宇航员 + 无国旗臂章 + 高反差 + 吸睛`

### 4. 前端展示

在新增候选稿的最终结果卡片中，增加一个独立展示块：

- 标题：`封面图 Prompt`
- 文本内容：完整 prompt
- 操作：`复制封面图 Prompt`

展示要求：

- 放在正文、标签、生成说明之后
- 样式和“复制发布稿”保持一致，但视觉层级更轻
- 不增加额外弹窗，不打断现有生成结果阅读

### 5. 数据结构

候选稿对象新增字段：

- `coverImagePrompt`

评分与修复流程中该字段遵循以下原则：

- 如果模型返回了有效值，则保留
- 如果修复轮没有返回该字段，则沿用原候选稿已有值
- 如果最终未生成成功，则显示本地规则兜底 prompt

### 6. 兼容性

现有发布稿复制逻辑不变。

新增的封面图 prompt 不参与：

- 生命周期保存
- 参考样本入库
- 误报记录
- 分数排序逻辑

它只服务于“新增候选稿”这一结果呈现与复制场景。

## 错误处理

- 如果模型返回空值或明显模板化废话，直接回退到本地规则 prompt
- 如果 prompt 过于空泛，优先补充标题和正文中的主题锚点
- 如果短文/长文模式规则冲突，以模式硬性规则优先

## 测试范围

- 新增候选稿结果对象包含 `coverImagePrompt`
- 短文模式 prompt 明确包含“不露脸的萌系宇航员”与“封面文案”
- 长文模式 prompt 明确包含“露脸的萌系宇航员”
- 两种模式都明确包含“去掉手臂国旗标识”
- 前端结果卡片展示 `封面图 Prompt` 与复制按钮
- 模型失败时会回退到本地规则 prompt
