# 学习样本反哺校验与生成设计

## 背景

当前系统中的学习样本已经承担了多项基础能力：

- 作为样本库内容沉淀、去重和编辑对象
- 作为生成与改写时的参考样本来源
- 作为风格画像与生命周期分析的输入

但在“内容校验”链路里，学习样本还没有真正参与运行时判定。当前校验主要依赖：

- 白名单
- 规则词库
- 误报日志
- 组合风险规则

用户希望把学习样本真正接入系统，让它们同时反哺：

1. 内容生成 / 改写
2. 内容校验

同时用户也明确要求不要把所有样本一股脑接入判定，避免误判与口径混乱。

## 目标

建立一套清晰、可扩展的样本分层机制，让不同质量、不同状态的学习样本在系统中承担不同角色：

1. 优质正向样本同时反哺生成和内容校验
2. 普通样本继续承担沉淀、去重、检索与人工筛选价值
3. 负向样本先保留结构与入口，后续再用于避坑与风险解释

本次只让“参考样本池”真正进入运行时正向生效链路，其余样本先完成分层约束，不直接参与判定。

## 非目标

本次不做以下能力：

- 用样本直接替代现有规则引擎
- 让样本参与 `hard_block` 放行
- 用普通样本或负向样本直接放宽校验结果
- 引入向量数据库或复杂语义召回
- 自动根据样本训练规则权重
- 自动把所有高数据样本直接升为参考样本

本次优先解决“优质正向样本如何稳妥参与系统”的问题。

## 样本分层

### 1. 参考样本池

进入条件：

- `reference.enabled === true`
- 数据达标
- 内容状态为正向合规

参考样本池是当前唯一可以直接反哺系统运行时行为的样本层。

当前作用：

- 参与生成
- 参与改写
- 参与内容校验中的“安全提示 / 轻微减误报”
- 参与风格、标签、集合类型和蜜语术语参考

### 2. 普通样本池

定义：

- 已导入样本库
- 但未启用参考，或数据未达标
- 内容本身仍属于正常沉淀素材

当前作用：

- 存档
- 去重
- 检索
- 人工筛选
- 作为后续晋升为参考样本或反例样本的候选

普通样本当前不直接参与生成与内容校验。

### 3. 反例样本池

定义：

- 违规
- 限流
- 低质量
- 或后续人工明确标记为不建议复用

当前阶段只保留结构和流转入口，不直接参与本次正向运行时能力。

后续预留用途：

- 改写避坑提示
- 校验风险解释
- 历史风险样本对照

## 参考样本池准入规则

参考样本池必须同时满足两个维度：

1. 人工认可
   条件：`reference.enabled === true`
2. 表现达标
   条件：样本达到预设的数据门槛

也就是：`手动启用参考 + 数据达标`，两者都满足才进入参考样本池。

之所以这样设计，是为了避免两类问题：

- 只靠人工启用，可能把效果一般的样本也放进运行时参考
- 只靠数据门槛，可能把用户并不认可的样本自动放进参考池

### 数据达标的口径

本次实现中，数据达标采用现有系统中可解释、可落地的固定门槛口径，并统一在一个辅助判定函数中维护。该门槛用于判断样本是否可以进入“参考样本池”，不直接改变样本原始数据。

## 生成链路中的生效方式

学习样本在生成 / 改写中的使用策略调整为：

1. 优先读取参考样本池
2. 普通样本不默认进入运行时生成参考
3. 反例样本不作为正向生成参考

生成时参考样本主要影响：

- 标题风格
- 正文结构
- 标签习惯
- 集合类型下的内容表达
- 蜜语术语的使用边界与风格

生成链路继续保持“参考而不照搬”的原则。

## 内容校验链路中的生效方式

### 总体原则

内容校验仍以现有规则链路为主。学习样本不会变成新的硬规则来源，也不会直接替代当前规则引擎。

本次新增的是一个位于规则命中之后的“参考样本提示层”。

校验顺序保持为：

1. 白名单
2. 规则词库
3. 误报日志
4. 组合风险规则
5. 参考样本提示层
6. 最终建议输出

### 参考样本提示层的边界

1. 只使用参考样本池中的样本
2. 只做“减误报”，不做“提风险”
3. 只允许有限放宽
4. 只在软风险场景中生效
5. 输出必须可解释

### 有限放宽规则

参考样本提示层只允许在以下场景参与调整：

- 当前结果是 `manual_review`
- 当前命中中不存在 `hard_block`
- 当前内容和参考样本池中的样本形成足够强的安全支持信号

满足条件时，允许把最终结果从 `manual_review` 放宽到 `observe`。

明确禁止：

- `hard_block` 被样本放宽
- 样本直接把结果改成 `pass`
- 普通样本或负向样本参与放宽

### 样本匹配策略

第一版不引入复杂召回系统，采用轻量、可解释的文本相似支持规则。

相似支持信号由以下维度组成：

1. 标题相似
2. 正文相似
3. 标签辅助相似
4. 集合类型辅助相似

最终形成一个 `sampleSupportScore`，并返回命中的样本列表与命中原因。

### 匹配保护条件

为避免“沾边就像”，样本提示层必须增加保护约束：

1. 标题或正文过短的样本不参与匹配
2. 当前内容命中明显硬风险时直接跳过样本放宽
3. 仅返回少量最相关样本，避免提示污染
4. 样本匹配结果必须能解释其支持来源

## 前后端输出要求

内容校验结果中新增以下样本相关字段：

- `referenceSampleHints`
- `matchedReferenceSamples`
- `referenceSampleSupportScore`
- `softenedByReferenceSamples`

页面上需要把这部分明确展示为“参考样本提示”，而不是“规则命中”。

展示文案应清楚说明：

- 当前是否命中相似参考样本
- 参考样本是否参与了轻微放宽
- 哪些样本提供了支持
- 为什么这些样本被认为相似

## 流转时机

### 样本导入后

样本刚导入时默认属于普通样本池，只承担：

- 存档
- 去重
- 运营记录
- 人工查看

此时不自动参与生成与内容校验。

### 升级为参考样本后

当样本同时满足：

- `reference.enabled === true`
- 数据达标

才进入参考样本池，并开始同时参与：

- 生成 / 改写
- 内容校验提示层

### 反例样本

本次只预留流转结构，不进入正向运行时链路。

## 测试要求

至少覆盖以下场景：

1. 参考样本池筛选
   仅 `手动启用参考 + 数据达标 + 正向合规` 的样本可参与生成与校验。

2. 生成样本筛选
   普通样本与负向样本不会进入正向参考样本集合。

3. 校验软放宽
   当结果为 `manual_review` 且命中相似参考样本时，可从 `manual_review` 降为 `observe`。

4. 硬风险保护
   当结果含 `hard_block` 时，参考样本提示层不得放宽结果。

5. 无匹配不生效
   当不存在足够强的参考样本支持信号时，最终结果保持不变。

6. 输出可解释
   校验结果包含参考样本提示字段、命中样本和支持分值。

## 验收标准

完成后应满足：

1. 学习样本被清晰分为参考样本、普通样本和反例样本三层角色
2. 参考样本池的准入口径明确为 `手动启用参考 + 数据达标`
3. 生成与改写只使用参考样本池作为正向运行时参考
4. 内容校验新增参考样本提示层
5. 参考样本只能把 `manual_review` 轻微放宽为 `observe`
6. `hard_block` 永远不被参考样本放宽
7. 校验结果对样本生效原因具备可解释输出
