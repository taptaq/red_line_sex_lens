# 成功样本驱动的自进化生成闭环设计

## 背景与目标

当前系统已经具备本地规则检测、语义复判、交叉复判、合规改写、违规反馈回流、误报样本回流和白名单候选复核能力。下一阶段目标是把这些能力连接成一个会持续学习的内容生产闭环：系统不仅判断一篇小红书笔记是否有风险，还能基于历史成功样本和账号风格画像，快速生成一篇更可能一次过审、且符合用户要求和风格的笔记。

本阶段不追求自动绕过平台审核，也不自动放行高风险内容。系统仍以合规、安全、人工可控为边界，重点提升“安全表达学习”“风格一致性”和“生成候选稿筛选”。

## 范围

本设计覆盖第一阶段推荐版能力：

- 成功发布样本库
- 账号风格画像
- 从零生成和草稿优化工作台
- 成稿评分与推荐
- 发布结果回流闭环

本阶段暂不加入转化线索、自动 A/B 实验、自动规则权重训练或自动发布能力。

## 成功样本库

新增成功样本数据，用于记录平台实际通过且可学习的正样本。样本支持两种入口：

- 手动录入历史成功笔记
- 从当前检测结果或改写结果一键保存

成功样本分为三档：

- `passed`：仅过审。用于学习安全表达和降低误杀。
- `performed`：过审且表现好。用于学习内容结构、标题策略和表达力度。
- `featured`：人工精选标杆。作为最高权重样本，生成时优先参考其风格。

第一版表现字段包括：

- 点赞数
- 收藏数
- 评论数
- 发布时间
- 人工备注

样本主体字段包括标题、正文、封面文案、标签、来源、保存入口、关联检测快照、关联改写快照和创建/更新时间。后续所有学习和生成引用都应保留来源信息，避免系统给出无法追溯的建议。

## 风格画像

风格画像从 `featured` 和 `performed` 样本中自动总结，但必须经过人工确认或编辑后才生效。系统可以生成“待确认画像”，用户确认后成为当前生效画像。

画像内容包括：

- 标题风格：标题长度、常用句式、是否口语化、是否偏科普或种草
- 正文结构：开头方式、段落节奏、信息密度、是否使用清单
- 语气与人设：专业、亲和、克制、陪伴感、经验分享感等
- 标签偏好：常见标签和不建议使用的标签
- 表达边界：不符合用户风格或容易触发风险的表达
- 生成要求：默认内容方向、禁用语气、保留程度

画像生成时应显示引用了哪些样本，并允许用户编辑每一项。未确认画像不能影响正式生成，最多作为预览建议。

## 生成与优化工作台

工作台支持两种模式：

- 从零生成：用户输入主题、卖点、目标人群、注意事项和标签偏好，系统输出完整小红书笔记。
- 草稿优化：用户粘贴已有标题、正文、封面和标签，系统改成更安全、更符合风格的版本。

生成过程默认输出多个候选稿，而不是单稿。第一版建议生成三类候选：

- 安全保守版：优先降低审核风险。
- 自然口语版：优先贴近账号风格和阅读体验。
- 表达增强版：在安全边界内提高吸引力和完整度。

每个候选稿都应包含标题、正文、封面文案、标签、生成说明、安全注意事项和引用的成功样本摘要。引用成功样本时只作为风格和结构参考，不照搬原文。

## 成稿评分

每个候选稿生成后进入统一评分流程：

- 规则检测：复用本地规则、词库、白名单和误报降权提示。
- 语义复判：使用当前可选模型链路判断隐含风险。
- 交叉复判：使用独立复判模型组检查最终一致性。
- 风格匹配评分：对照当前生效风格画像和高权重成功样本。
- 内容完整度评分：检查主题、卖点、注意事项和标签是否被覆盖。

最终展示推荐稿，并说明推荐原因。推荐原因应包含合规风险、风格匹配、内容完整度和剩余人工注意点。若所有候选都仍为 `manual_review` 或更高风险，不应强行推荐“可直接发布”，而应提示继续人工调整。

## 自进化闭环

发布后用户可以把结果回填为以下类型：

- 成功样本：用于强化安全表达和风格画像。
- 误报样本：用于降权提示和宽松白名单候选。
- 违规反馈：用于候选风险词和规则复核。
- 改写前后样本：用于评估哪些改写策略有效。

系统不直接自动修改正式词库、白名单或生效风格画像。所有会影响后续检测或生成的学习结果，都进入候选状态并由人工确认后生效。

## 数据流

1. 用户输入主题或草稿。
2. 系统读取当前生效风格画像、成功样本、误报样本、白名单、词库和规则。
3. 系统检索相似成功样本和相关反例样本，生成多个候选稿。
4. 每个候选稿自动跑检测、复判和评分。
5. 页面展示推荐稿、备选稿、风险原因、风格评分和引用依据。
6. 用户发布后回填平台结果。
7. 平台结果进入成功样本、误报样本、违规反馈或改写样本库。
8. 新样本继续生成候选规则、候选白名单和候选风格画像。

## 人工控制与安全边界

- `hard_block` 不因成功样本、误报样本或风格画像直接放行。
- 成功样本只提供正向参考，不自动覆盖规则判断。
- 风格画像必须人工确认后才生效。
- 白名单和词库变更必须经过人工复核。
- 成稿评分必须显示来源和模型尝试信息，避免“黑箱推荐”。

## 第一阶段验收标准

- 可以手动新增成功样本，也可以从当前检测/改写结果一键保存。
- 成功样本支持 `passed`、`performed`、`featured` 三档。
- 可以从高权重成功样本生成待确认风格画像。
- 已确认风格画像会参与生成提示词。
- 生成工作台支持从零生成和草稿优化。
- 候选稿会自动经过规则检测、语义复判、交叉复判和风格评分。
- 页面会推荐最优候选，并解释推荐原因。
- 发布后可以把结果回填为成功样本，形成下一轮学习输入。

## 测试策略

- 数据层测试成功样本的新增、更新、去重和分档保存。
- API 测试手动录入、一键保存、画像生成、画像确认和生成请求。
- 生成流程测试候选稿会携带模型选择、风格画像和样本引用。
- 评分流程测试风险结论、风格评分和推荐排序。
- 前端源码测试页面入口、保存按钮、画像确认面板和推荐结果展示。
- 回归测试确保误报样本降权、白名单候选、模型选择和现有改写流程不被破坏。
